# UI Rules

## Product direction

- Mobile-first for the member experience
- Clean, calm, high-trust visual language
- Large touch targets and clear primary actions
- Make safety and readiness visible before actions are taken

## Mobile app

- Keep onboarding and queue screens simple
- Use strong hierarchy for readiness, match preview, and next-step guidance
- Prefer one clear next action per card
- Match signals should feel explanatory, not noisy

## Web admin

- Make moderation data dense but scannable
- Use clear severity states for health, SLA, and backlog
- Keep bulk actions visible but gated when health degrades
- Surface queue pressure, member detail, and exports without extra navigation

## Accessibility

- Large fonts
- High contrast
- Easy onboarding
- Clear labels for every action and status
